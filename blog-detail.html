Laravel Performance Optimization with Eloquent

LaravelPerformanceDatabase

Saat mengembangkan aplikasi dengan Laravel, salah satu isu performa yang paling sering muncul adalah N+1 Query Problem. Masalah ini biasanya terjadi saat kita me-load model beserta relasinya menggunakan loop tanpa eager loading yang pada akhirnya menyebabkan ratusan query database tereksekusi tanpa disadari.

1. Menggunakan Eager Loading

Eager loading adalah cara paling dasar dan efektif untuk mengatasi masalah N+1. Daripada me-load relasi saat dibutuhkan di dalam loop, kita bisa memuat semuanya di awal menggunakan method with().

User::with('posts')->get();

Namun, seringkali kita tidak membutuhkan semua data dari tabel yang berelasi. Di sinilah Advanced Eager Loading sangat berguna untuk menghemat penggunaan memori.

2. Eager Loading Spesifik Kolom

Jika tabel relasi Anda memiliki banyak kolom, mengambil semua datanya bisa membebani memori. Anda bisa secara eksplisit memilih kolom mana yang akan di-load pada relasi.

User::with('posts:id,user_id,title')->get();

3. Caching Query yang Sering Diakses

Meskipun query sudah dioptimalkan seminimal mungkin, memanggil database secara terus menerus tetap akan memakan resource (seperti CPU dan latency jaringan). Untuk data yang jarang berubah namun sering diakses, menggunakan Redis/Memcached sebagai Cache adalah solusi terbaik.

$users = Cache::remember('users_with_posts', 3600, function () {
    return User::with('posts')->get();
});

Kesimpulan

Optimasi database di ekosistem Laravel bukanlah proses sekali jalan, melainkan iterasi berkelanjutan seiring dengan pertumbuhan jumlah data pengguna. Dengan memahami kumpulan alat yang disediakan oleh Eloquent ORM, Anda dapat memastikan aplikasi berjalan responsif, bahkan dengan jutaan baris data.
